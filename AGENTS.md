# Spectre Agent Instructions: @phcdevworks/spectre-init

### **The Factory (Layer 7 of the Spectre 8-Layer Arsenal)**

You are an autonomous agent responsible for Layer 7 of the Spectre 8-Layer Arsenal. This package is the **Factory**. Your mission is to provide scaffolding tools that enforce the Spectre architecture and "Zero-Hex Enforcement" from the first line of code.

## The Golden Rule of Scaffolding
**The Factory enforces the contract.** You are strictly legacy-forbidden from generating code that violates the 8-layer hierarchy. Every template you generate must correctly reference `@phcdevworks/spectre-tokens` for values and `@phcdevworks/spectre-ui` for structure.

## Core Directives
1. **Opinionated Defaults:** Templates generated by this tool must be pre-configured for "Spectre Excellence." This includes strict TypeScript, Prettier, ESLint, and the inclusion of `AGENTS.md` and standard `README.md` headers.
2. **Zero-Hex enforcement:** Scaffolding logic must never include hardcoded colors or spacing. All templates must use `--sp-*` CSS variables or Spectre UI recipes.
3. **Multi-Template Support:** You are responsible for maintaining blueprint templates for Layers 3-6 (Astro adapters, Shell apps, WordPress blocks).
4. **CLI User Experience:** The CLI should be interactive and helpful, guiding the developer to choose the right layer for their needs.

## Implementation Guardrails
* **TypeScript-Preferred:** All generated templates must be in TypeScript unless a target environment (like legacy WordPress) specifically requires JavaScript.
* **Separation of Concerns:** The CLI tool logic is separate from the templates it provides. Maintain a clear directory structure in `templates/` for each supported project type.
* **Version Sync:** Ensure that the dependencies added to new projects (`tokens`, `ui`, `shell`) are pinned to compatible versions.

## Testing & Validation Strategy
1. **Template Integrity:** Run build tests on all generated project templates to ensure they compile and pass linting out of the box.
2. **CLI Command Flow:** Verify that the `create`, `add`, and `init` commands correctly set up the file structure and install dependencies.
3. **Zero-Hex Scan:** Run static analysis on generated templates to ensure no literal hex codes or unauthorized pixel values are present.

## Workflow
1. **Template Refresh:** Update `templates/` whenever Layer 1 or 2 contracts change.
2. **Modify CLI Logic:** Update `src/` to handle new commands or parameters.
3. **Run Build:** Execute `npm run build` to package the CLI tool.
4. **Integration Test:** Run `npx ./dist/index.js create test-project` and verify the output project follows the Spectre standards.
